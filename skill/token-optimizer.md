name: token-optimizer
description: Оптимизация стоимости subagent-вызовов через правильный выбор модели haiku/sonnet/opus. Вызывается командами /model или /token-optimizer.
---

Когда пользователь вызывает этот скилл, выполни следующие шаги по порядку:

## Шаг 1 — Показать таблицу роутинга моделей

Выведи таблицу выбора модели для subagent-вызовов:

| Тип задачи | model= | Примеры |
|---|---|---|
| Разведка: grep/read/find/SQL/инвентаризация | `"haiku"` | Explore, поиск файлов, SQL-запросы |
| Технический анализ: парсинг, форматирование, сравнение данных | `"haiku"` | audit, extract, cavecrew-investigator |
| Сложный исполнитель: написание кода, рефакторинг | `"sonnet"` | builder-*, webdev-*, coder |
| Мышление, синтез, стратегия, общение с пользователем | основной контур (не subagent) | Сам Claude в CLI |
| Opus 4.8 | только через Max-подписку | Opus 4.8 → ТОЛЬКО через Claude Max-подписку (claude-cli-client.mjs). Через API = $75/M output, запрещено правилом. |

**Правило:** перед каждым `Agent(...)` ОБЯЗАТЕЛЬНО указывать `model=`. Без `model=` — нарушение, утечка бюджета.

> ⚠️ Лимит: максимум 10 параллельных subagent одновременно. Тяжёлые (crawl/playwright/build) = вес 2, сумма весов ≤ 10.

---

## Шаг 2 — Запросить код для аудита

Попроси пользователя показать файл или фрагмент кода, где запускаются `Agent()` вызовы. Спроси:

- Путь к файлу(ам) с `Agent(` вызовами
- Примерное количество вызовов в день (если известно)
- Текущий месячный бюджет на API (если известно)

---

## Шаг 3 — Найти все Agent() без model=

Когда получишь путь к файлу, выполни поиск:

```bash
grep -rn "Agent(" . | grep -v "model=" | grep -v "node_modules"
```

Каждый найденный вызов — потенциальная потеря бюджета. Составь список:
- Номер строки
- Фрагмент кода
- Рекомендуемая модель (haiku/sonnet по таблице выше)

---

## Шаг 4 — Рассчитать экономию

Используй приблизительные цены (за 1M токенов):

| Модель | Input | Output |
|---|---|---|
| claude-haiku-4-5 | $0.80 | $4.00 |
| claude-sonnet-4-5 | $3.00 | $15.00 |
| claude-opus-4-8 через API | $15.00 | $75.00 (ЗАПРЕЩЕНО правилом) |
| claude-opus-4-8 через Max | $0 (подписка) | $0 (подписка) |

Формула для одного вызова (средний размер):
- Haiku: ~$0.002 на вызов (1K input + 500 output токенов)
- Sonnet: ~$0.010 на вызов
- Opus API: ~$0.050 на вызов

> Средний вызов = ~2000 input + ~500 output токенов (для задач разведки/grep/SQL). Для кодинга умножь на 3-5x.

**Расчёт экономии в $/мес:**
```
Экономия = (N_вызовов_в_день × 30) × (стоимость_sonnet - стоимость_haiku)
```

Пример: 50 разведочных вызовов/день на sonnet вместо haiku:
- Sonnet: 50 × 30 × $0.010 = $15/мес
- Haiku: 50 × 30 × $0.002 = $3/мес
- **Экономия: $12/мес (~80%)**

## Шаг 5 — Показать конкретный diff для исправления

Для каждого найденного нарушения покажи точное исправление в формате diff:

```diff
- Agent({
-   description: "Поиск файлов конфигурации",
-   prompt: "Найди все .env файлы в /root/..."
- })
+ Agent({
+   description: "Поиск файлов конфигурации",
+   prompt: "Найди все .env файлы в /root/...",
+   model: "haiku"
+ })
```

Для каждой задачи обоснуй выбор модели одной строкой.

## Дополнительные правила (напомнить пользователю)

1. **Opus через API запрещён** для рутинных задач — только через Max-подписку (`claude-cli-client.mjs`)
2. **Sonnet** — только когда реально нужно написать/отрефакторить код, не для чтения
3. **Haiku** — дефолт для всего, что "посмотреть", "найти", "проверить", "сравнить"
4. **Основной контур** (Claude без subagent) — для синтеза, стратегии и финального доклада пользователю
5. **Лимит параллельных subagent-ов:** не более 10 одновременно; тяжёлые задачи (crawl/playwright/build) имеют вес 2

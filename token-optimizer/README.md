# Token Optimizer для Claude Code


Скилл учит Claude Code правильно выбирать модель для каждого subagent-вызова.
Результат — снижение расходов на API в 3–5 раз без потери качества.

---

## Проблема

Когда запускаешь subagent без явного `model=`, Claude Code по умолчанию берёт **sonnet**.

Sonnet тарифицируется примерно в 10× дороже haiku. Если у тебя 20–50 subagent-вызовов в день на grep, SQL-запросы, поиск файлов — всё это прожигает бюджет впустую.

**Реальный пример:** агентство AiPepDen использовало sonnet для всех subagent-ов.
Итог — **$65 в неделю** только на разведывательные задачи, которые haiku решает за $6.

---

## Решение: 3-уровневый роутинг

| Уровень | Модель | Когда использовать | Примерная стоимость |
|---------|--------|-------------------|---------------------|
| **Разведка** | `haiku` | grep, find, SQL SELECT, чтение файлов, инвентаризация, поиск паттернов | ~$0.25 / 1M tokens input |
| **Исполнение** | `sonnet` | написание кода, рефакторинг, сложный анализ, генерация | ~$3 / 1M tokens input |
| **Синтез** | основной контур | стратегия, финальные решения, общение с пользователем | — |
| **Opus** | Max-подписка | глубокое мышление, сложные архитектурные решения | $0 через API (только Max) |

### Правило одной строки

```
разведка/grep/SQL → haiku | код/рефакторинг → sonnet | мышление → основной контур
```

## Использование

После установки в любом проекте:

```
/model
```

Скилл анализирует задачу и выдаёт рекомендацию с объяснением — какую модель брать и почему.

### Примеры вызовов в коде

```js
// Разведка — haiku
Agent({ description: "Найти файлы конфигурации", model: "haiku", prompt: "..." })

// Исполнение — sonnet
Agent({ description: "Написать миграцию БД", model: "sonnet", prompt: "..." })

// Параллельная волна — явно указывай model для каждого
Agent({ model: "haiku", prompt: "grep по логам..." })
Agent({ model: "haiku", prompt: "SQL SELECT по таблице..." })
Agent({ model: "sonnet", prompt: "написать обработчик..." })
```

# Session Handoff — умная замена /compact для Claude Code

Скилл для Claude Code, который сжимает текущую сессию в структурированное саммари вместо `/compact`. После `/clear` вставь саммари — и продолжай с того же места без потери контекста.

---

## Проблема

`/compact` теряет нюансы и не всегда справляется с длинными сессиями. Следующая сессия начинает с нуля: нет понимания что сделано, какие файлы трогали, где остановились.

Кэш промпта в подписке живёт **1 час**, у субагентов — **5 минут**. Раздутая сессия = дорогой пересчёт кэша при каждом сообщении.

## Решение

Скилл `/session-handoff` автоматически:

1. Собирает саммари по структуре (цель → сделано → файлы → развилки → следующий шаг)
2. Записывает файл в `/root/sokrat-reports/handoffs/`
3. Выводит блок в чат для копирования
4. Обновляет `CLAUDE.local.md` — секцию `## Последняя сессия`

После `/clear` вставляешь саммари — и Claude сразу понимает контекст.

## Вызов

```
/session-handoff
```

Также срабатывает на: «передай сессию», «хэндофф», «сверни сессию», «продолжим с чистого».

Автоматически предлагается когда:
- контекст забит >70%
- сессия идёт >1 часа
- переключаемся на другую задачу

## Установка

```bash
# Клонируй репозиторий или скачай папку session-handoff/
bash install.sh /путь/к/проекту
```

Скрипт скопирует `skill/session-handoff.md` в `.claude/skills/` проекта.

После установки перезапусти Claude Code — скилл появится автоматически.

## Пример саммари

```markdown
# HANDOFF: Починка Надзора — 2026-06-21 14:30

## Цель сессии
Починить 5 критичных инцидентов Надзора, мигрировать .env на supabase-db:5432

## Сделано
- Надзор починен, паспорт обновлён (mode='active')
- 6 .env мигрированы с baza.aipepden.store → supabase-db:5432

## Файлы затронуты
- /root/novu-clone/agents/anna-director/.env — обновлён DSN

## СЛЕДУЮЩИЙ ШАГ (continue here)
Проверить smoke-команды всех 6 воркеров: `pm2 ls | grep -E 'anna|gena'`

## Не забыть
- pm2 --update-env из Bash-сессии инжектит .env хэрнесса → EADDRINUSE краш
```

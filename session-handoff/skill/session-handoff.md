# session-handoff — передача сессии без потери контекста

## Зачем (механика кэша, перенято у Nate Herk)

- Кэш промпта в подписке живёт **1 час**, у субагентов/API — **5 минут**. Долгая раздутая сессия = дорогой пересчёт.
- `/compact` иногда долгий и теряет нюансы. `/clear` + готовое саммари — быстрее и точнее: ты сам решаешь, что важно сохранить.
- Цель: свернуть смысл сессии в компактный markdown, после `/clear` вставить его — и продолжить как ни в чём не бывало.

## Что делать

1. **Собери саммари текущей сессии** по структуре ниже. Бери из ФАКТИЧЕСКОГО хода работы, не выдумывай.
2. **Запиши в файл** `/root/sokrat-reports/handoffs/<тема>-<дата>.md` (создай папку, если нет) — чтобы не потерялось, даже если чат закроют.
3. **Выведи саммари прямо в чат** компактным блоком, чтобы можно было скопировать.
4. **Скажи**: «Готово. Делай `/clear` и вставь этот блок — продолжим с того же места. Файл: <путь>».
5. **Обнови `CLAUDE.local.md`** — замени секцию `## Последняя сессия` в `/root/novu-clone/CLAUDE.local.md` через Edit tool:
```markdown
## Последняя сессия

- **Дата:** {YYYY-MM-DD}
- **Тема:** {краткое описание из «Цель сессии»}

### Что сделали:
{bullet-list из секции «Сделано» handoff}

### Открытые вопросы (TODO):
{список из «Открытые решения / развилки» handoff}
```

## Структура саммари

```
# HANDOFF: <тема> — <дата/время>

## Цель сессии
<1-2 строки: что вообще делаем и зачем>

## Сделано
- <конкретный результат + где лежит>
...

## Файлы затронуты
- <путь> — <что изменено/создано>
...

## Открытые решения / развилки
- <вопрос, который ещё не решён, и варианты>
...

## Что осталось (открытые задачи)
- <незакрытая задача + приоритет>
...

## Следующий приоритет
<конкретная первая задача следующей сессии>

## Состояние инфры/БД (если релевантно)
- <что задеплоено, какие воркеры/таблицы, что проверено смоуком>

## Технические артефакты
- PM2: <новые/изменённые процессы>
- Таблицы: <новые таблицы БД>
- Файлы: <ключевые новые файлы>

## СЛЕДУЮЩИЙ ШАГ (continue here)
<точная первая команда/действие для новой сессии — максимально конкретно>

## Не забыть
- <грабли, ловушки, что НЕ делать>
```

## Правила

- **`CLAUDE.local.md` обновляется ОБЯЗАТЕЛЬНО.** Шаг 5 не опциональный — Edit секцию `## Последняя сессия` напрямую (не echo/sed/awk). Без этого шага handoff неполный.
- **Только факты сессии.** Что реально сделали, какие файлы трогали, что решили — не пересказ общих знаний.
- **«Следующий шаг» — конкретный.** Не «продолжить работу», а «дописать чекпойнт в plan-roaster, затем запустить grill-me по ГенКоду».
- **Ловушки обязательны.** Если по ходу сессии напоролись на грабли — вынести в «Не забудь».
- **Дубль в память — по запросу.** Если сессия значимая, предложи занести итог в MEMORY.md — но это отдельный шаг, не путать с хэндоффом.

## Анти-паттерны

- Лить весь транскрипт — это не сжатие.
- Размытый «следующий шаг».
- Забыть записать файл (только в чат) — потеряется при закрытии.
